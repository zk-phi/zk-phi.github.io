麻雀メモ

グループを見つける

大前提として、どんな牌でも、好きな牌をあと２枚持ってこれるなら３枚ブロックになる。

 例： 1s は、 2s と 3s を持ってこれるなら３枚ブロックになる

逆にいえば、「あと２枚くればいい感じになる」は何も揃ってないのと同じ。

したがって、すべての牌は「A. すでに３枚ブロックになってる牌」「B. あと１枚で３枚ブロックになれそうな牌」「C. あと２枚こないと３枚ブロックになれない牌（浮き牌）」のどれかに必ず属する。

 例： 1m, 3m, 6m, 9m, 1p, 1p, 2p, 5p, 1s, 2s, 3s, 7s, 7s を持ってる場合
  (1s, 2s, 3s) は「A. すでに３枚ブロック」
  (1m, 3m), (1p, 1p), (1p, 2p), (7s, 7s) は「B. あと１枚で３枚ブロック」
   ※ 1p は２つのグループに属している
  6m, 9m, 5p は「浮き牌」

麻雀の上がり形は「A を４つ、 B を１つ、の計５ブロック」なので、

 浮き牌を育てて B を作る
 B を育てて A を作る

のように牌を育てていくことで上がりが近づいてくる（シャンテン数が減ってくる）ゲームになっている。

牌の切り方

上の仕組みを考えると、最速でテンパイまでたどり着く（シャンテン数を単調減少させる）にはざっくり以下を考えればいい。たぶん。

 A と B 合わせてすでに５セットあるなら、浮き牌は捨てていく
 A と B 合わせて６セット以上あるなら、育ちにくそうな B から捨てていく
 逆に A と B 合わせて５セットに満たないなら、どこかに B をもう１つ作る必要がある

実際には点を稼ぐために、多少回り道をしてもドラを育てるとか、そういうケースはありそう。あるいは回り道をした方が結果的に早く上がれるケースとかもあるかも？

B グループの分類

B グループは必ず次のどれかの形になる。

両端のどっちかがくると階段になる（リャンメン）
 例： (2s, 3s) → 1s か 4s で揃う
 受け入れが一番広い、最強

片端がくると階段になる（ペンチャン）
 例： (1s, 2s) → 3s で揃う
 ドンピシャが来ないと揃わないし、タンヤオにもならないし、弱そう

真ん中が来ると階段になる（カンチャン）
 例： (3s, 5s) → 4s で揃う
 ドンピシャが来ないと揃わないけど、 2s か 6s が来るとリャンメンに持ち変えられるのでペンチャンよりマシっぽい

同じのが来るとスリーカードになる（シャンポン）
 例： (2s, 2s) → 2s で揃う
 ４枚ある 2s のうち２枚をすでに持ってるところで、さらに 2s をもう一枚引いてこないといけないので相当厳しい
  ポン、ロンでもらってくるのが現実的
 手札に１〜２セットあるのがちょうどいい
  ひとつもない場合 → テンパイしたときジャントウを待つ形になってしんどい
  １セットある場合 → ジャントウとして使える
  ２セットある場合
   ２セットあるうちのどちらかをスリーカード、余った方をジャントウに使えば良い
    例： (1p, 1p), (2s, 2s) → 1s がきても 2s がきても２枚ペア＋スリーカードで上がり
  ３セットある場合
   うち１セットは自力で揃えないと (あるいはポンしないと) テンパイまでいけないのでしんどい

基本的に、揃いにくそうな形の B より、揃いやすそうな形の B を手札に残す方が早く上がれる可能性が高い (牌効率)。ただし、待ってる牌がすでに場に捨てられてる場合などは別。たとえば (2s, 3s) のリャンメンでも、 1s がすでに場に 4 枚捨てられてるなら、 4s しか期待できないからあんまり強くない。

同様に、弱い B に育ちやすそうな浮き牌 (1, 2, 8, 9, 字牌) より、強い B に育ちやすそうな浮き牌を手札に残す方が早く上がれる可能性は高い。

２つの B が１つの牌を共有する形

３枚で１ブロックを待つことになるので物理的にかさばるけど、受け入れは広がる。

カンチャン＋カンチャン（リャンカン）
 例: (2s, 4s, 6s) ... (2s, 4s) と (4s, 6s) がカンチャン → 3s か 5s がくれば揃う
 どっちも自分は持っていない牌なので、山に各 4 枚づつ、計 8 枚ある。強い。

以下はシャンポンが絡むので、リャンカンよりはちょっと弱い。

カンチャンの一端がシャンポン
 例： (2s, 2s, 4s) ... (2s, 2s) がシャンポン、 (2s, 4s) がカンチャン → 2s か 3s がくれば揃う

リャンメンの一端がシャンポン
 例： (2s, 2s, 3s) ... (2s, 3s) がリャンメン、 (2s, 2s) がシャンポン → 1s か 2s か 4s がくれば揃う

※リャンメン＋カンチャンは、よく見ると受け入れが広がってない
 例： (2s, 3s, 5s) … (2s, 3s) リャンメンの時点で 1s, 4s を受け入れているので、 5s がついても意味ない (捨てられる)

見づらい A グループに注意

２つの A グループ (あるいは A グループと B グループ) がくちゃっとなってるやつ。パッと見でパターン認識できるようになりたい。

階段と階段
 例： (1s, 2s, 3s, 4s, 5s, 6s) → (1s, 2s, 3s) と (4s, 5s, 6s) が隣り合ってる
 例： (1s, 2s, 3s, 3s, 4s, 5s) → (1s, 2s, 3s) と (3s, 4s, 5s) が重なってる
 例： (1s, 2s, 2s, 3s, 3s, 4s) → (1s, 2s, 3s) と (2s, 3s, 4s) が重なってる
 例： (1s, 1s, 2s, 2s, 3s, 3s) → (1s, 2s, 3s) が２セット (イーペーコー)

階段とペア
 例： (1s, 2s, 3s, 4s, 4s) → (1s, 2s, 3s) と (4s, 4s) が隣り合ってる
 例： (1s, 2s, 2s, 2s, 3s) → (1s, 2s, 3s) と (2s, 2s) が重なってる

多面待ちを見抜く

「見づらい A グループ」に育つタイプの B グループ。わかりづらいけど受け入れが広くて強い形なので、見抜けるようになりたい。全体を２ブロックとして数える。

以下は特に覚えておいた方がいい形。らしい。

４個階段・５個階段をふた通りに使える形
 例： (2s, 3s, 4s, 5s, 6s)
  (2s, 3s, 4s) ＋ (5s, 6s) と見ると、 (5s, 6s) のリャンメン (4s, 7s 待ち)
  (2s, 3s) ＋ (4s, 5s, 6s) と見ると、 (2s, 3s) のリャンメン (1s, 4s 待ち)
  どっちが揃っても計２メンツできる
 例： (4s, 5s, 5s, 6s, 7s) ... ターツ＋１メンツ
  (4s, 5s, 6s, 7s) の４個階段に注目
  (4s, 5s, 6s) + 7s と見ると、 7s がくれば１メンツ＋１ジャントウ
  4s + (5s, 6s, 7s) と見ると、 4s がくれば１メンツ＋１ジャントウ
  (4s, 5s) + (6s, 7s) と見ると、
   (6s, 7s) のリャンメンと (5s, 5s) のジャントウ (5s, 8s 待ち、 4s は捨てる)
   メンツが２つ欲しい場合は、リャンメンが２つと見ることもできる
  １メンツ＋１ジャントウが作りやすい
 例： (2s, 4s, 5s, 6s, 6s, 7s) ... リャンメンカンチャン
  (4s, 5s, 6s, 7s) の４個階段に注目
  (4s, 5s, 6s) ＋ 7s と見ると、 6s, 7s のリャンメン (5s, 8s 待ち, 2s は捨てる)
  4s ＋ (5s, 6s, 7s) と見ると、 2s, 4s のカンチャン (3s 待ち, 6s は捨てる)
  どっちが揃っても計２メンツできる

スリーカードが (ペア＋１) としても使える形
 例： (2s, 2s, 2s, 3s, 4s, 9s, 9s)
  (9s, 9s) のシャンポンは孤立しているので置いておいて、 (2s, 2s, 2s, 3s, 4s) に注目
  (2s, 2s, 2s) + (3s, 4s) と見ると、 (3s, 4s) のリャンメン (2s, 5s 待ち、 (9s, 9s) は雀頭)
  (2s, 2s) + (2s, 3s, 4s) と見ると、 (2s, 2s) と (9s, 9s) がシャンポン (2s, 9s 待ち)
   2s がシャンポンであることで、受け入れが 9s まで伸びるのがミソ
  どれがきても必ずジャントウ＋２メンツできる ... (2s, 2s, 2s, 3s, 4s) 部分を２ブロックとして数える

リャンカンにシャンポンが重なった形
 例： (3s, 3s, 5s, 7s, 7s) ... サンドイッチ形
  (3s, 5s, 7s) のリャンカンに注目
  (3s, 5s) + 7s と見ると、 (3s, 5s) カンチャン ＋ (7s, 7s) シャンポン (4s, 7s 待ち, 3s は捨て)
  3s + (5s, 7s) と見ると、 (3s, 3s) シャンポン ＋ (5s, 7s) カンチャン (3s, 6s 待ち, 7s は捨て)
  どれがきても必ずジャントウ＋１メンツできる

一番弱い B を強くする

麻雀は４ブロックを A に育てないと上がれないゲームなので、「いい感じの B」よりも「ヤバい感じの B 」が律速要因になりやすい。

 例： 手札がパンパンで (2s, 2s, 3s) と (5p, 5p, 8p) のどちらかから１枚切らないといけない場面
  2s を切る → (2s, 3s) のリャンメンが残るので十分いけそう
  3s を切る → (2s, 2s) のシャンポンが残ってキツそう
  5p を切る → (5p, 8p) のカンチャンが残ってキツそう
  8p を切る → (5p, 5p) のシャンポンが残ってキツそう
  より強い形の (2s, 2s, 3s) から１枚切って、より弱い形の (5p, 5p, 8p) を守ってあげた方がトータル揃いやすい

4 グループしかない場合

手札に４グループしかない場合、どこかに１グループ生やす必要がある。

一番シンプルなのは孤立牌の周りの牌を引くこと。

 例： 3s → 1s か 5s を引くとカンチャン、 2s か 4s を引くとリャンメン、 3s を引くとシャンポン

だけど、「多面待ち」と同様に、もっと５グループ目が生えやすい形もある (くっつき１シャンテン)。

４階段 (ノベタン)
 例： (3s, 4s, 5s, 6s)
  (3s, 4s, 5s) + 6s と考えると、 6s の周りを引けば５グループ目が生える
  3s + (4s, 5s, 6s) と考えると、 3s の周りを引けば５グループ目が生える
  ジャントウを生やしたいときにも便利 (3s, 6s のどちらがきてもジャントウができる)

中ぶくれ３階段
 例： (3s, 4s, 4s, 5s)
  (3s, 4s) + (4s, 5s) と考えると、すでにリャンメンが２ブロックあるとも考えられる
  (3s, 4s, 5s) + 4s と考えると、 4s の周りを引けばそれを５グループ目として使うこともできる
  リャンメンになりやすいので強い

階段＋カンチャン
 例： (3s, 4s, 5s, 7s)
  (3s, 4s) + (5s, 7s) と考えると、すでにリャンメンとカンチャンが１ブロックずつあるとも考えられる
  (3s, 4s, 5s) + 7s と考えると、 7s の周りを引けばそれを５グループ目として使うこともできる

４ブロックしかないときにこういう形を見かけたら、孤立牌より優先して温存する。

浮き牌をあえて残す場合

すでに５ブロック揃っている場合、浮き牌はすべて捨ててしまってもテンパイできるが、あえて残すと強いケースもある：

 例： (1p, 2p, 3s, 4s, 5s, 6s)
  もし 3s, 6s の周りを引いた場合、 (1p, 2p) のペンチャンよりマシなブロックができて、入れ替えられる
